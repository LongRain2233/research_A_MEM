# 修改总览

## 4个Bug的快速对比

### Bug 1: Entailment逻辑反向
- **文件**: `renorm_engine.py` L196-198
- **原因**: premise=旧笔记, hypothesis=sigma → 问"小蕴含大"，逻辑反
- **修复**: 交换参数 → premise=sigma, hypothesis=旧笔记
- **影响**: theta_evict参数从无效→生效

### Bug 2: Global Decay无差别衰减
- **文件**: `sqlite_store.py` L133-155
- **原因**: 无WHERE条件，所有节点都被衰减
- **修复**: 添加WHERE过滤 → 不衰减abstract节点，保护1小时内活跃节点
- **影响**: Temporal/Multi-hop恢复（历史节点不被腐蚀）

### Bug 3: Abstract节点稀释检索
- **文件**: `orchestrator.py` search_with_graph() 方法
- **原因**: Abstract节点固有的"概括"特性导致具体细节丢失
- **修复**: 检索结果中abstract节点比例限制为30%
- **影响**: Multi-hop恢复（从17→26）

### Bug 4: 评测标准与A-MEM不统一
- **文件**: `metrics.py` 所有度量函数
- **问题**: F1(Counter vs Set), BLEU(无smoothing), ROUGE(无stemmer), SBERT(0-100)
- **修复**: 统一对齐A-MEM的计算方式
- **影响**: Open Domain从35→8-12（与原文对齐）

## 修改前后对比

| 方面 | 修改前 | 修改后 |
|------|--------|--------|
| **F1** | Counter重复词多计 | Set去重+标点处理 |
| **BLEU-1** | 自定义无smoothing | nltk + 长度惩罚 |
| **ROUGE** | use_stemmer=False | use_stemmer=True |
| **SBERT** | 0-100范围 | 0-1范围 |
| **Decay** | 无条件衰减所有 | 条件衰减：非abstract+1小时外 |
| **驱逐机制** | 无策略生效 | 正常工作 |
| **Abstract混入** | 无限制 | 最多30% |
