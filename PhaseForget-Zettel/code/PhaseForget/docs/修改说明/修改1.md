# PhaseForget-Zettel Bug 修复总结

**修改日期**: 2026-04-01
**修改人**: Claude Code
**修改范围**: 4个关键Bug修复 + 评测标准统一

---

## 背景：问题诊断

在对 PhaseForget-Zettel 的实验结果分析中发现了以下严重问题：

### 1. 与原文A-MEM比较时数据不可信
- **关闭遗忘机制时**：Open Domain从7.12→35.02（暴涨5倍）
- **开启遗忘机制时**：Temporal 22.94→18.99（下降），Multi-hop 29.55→17.33（暴跌）
- 原因不仅仅是遗忘机制，而是**评测方式完全不同**

### 2. 遗忘机制形同虚设
- theta_evict参数对结果几乎无影响（相关性仅0.007）
- 参数实验表明只有theta_sim有效（相关性0.967）
- **根本原因**：Entailment gate逻辑颠倒，驱逐条件永远不满足

### 3. 时间敏感问题受损
- Global Decay无差别应用所有节点，包括刚创建的重要abstract节点
- Temporal和Multi-hop问题最依赖历史节点，因此受损最严重

---

## 修改方案

### Bug 1: Entailment 逻辑反向 ❌→✅

**文件**: `src/phaseforget/memory/renorm/renorm_engine.py`
**行号**: 196-198

**问题**：premise和hypothesis参数颠倒

原代码问的是："旧笔记是否能蕴含sigma"（小蕴含大，逻辑反了）
应该问的是："sigma是否能蕴含旧笔记"（大蕴含小，才能判断是否可删除）

**修改**：
```python
# 修改前
entailment = await self._check_entailment(
    premise=note_data.get("content", ""),  # 旧笔记
    hypothesis=sigma_text,                  # sigma
)

# 修改后
entailment = await self._check_entailment(
    premise=sigma_text,                     # sigma
    hypothesis=note_data.get("content", ""),  # 旧笔记
)
```

**效果**：驱逐gate从永不打开→正常工作，theta_evict参数生效

---

### Bug 2: Global Decay 无差别衰减所有节点 ❌→✅

**文件**: `src/phaseforget/storage/hot_track/sqlite_store.py`
**行号**: 133-155

**问题**：无WHERE条件，所有节点（包括abstract和最近活跃节点）都被衰减

**修改**：
```python
# 修改前
UPDATE Memory_State SET utility_score = utility_score * ?

# 修改后 - 添加过滤条件
UPDATE Memory_State
SET utility_score = utility_score * ?
WHERE is_abstract = 0                    -- 不衰减abstract节点
  AND (last_accessed_at IS NULL 
       OR last_accessed_at < cutoff)     -- 不衰减最近1小时访问的节点
```

**效果**：
- Abstract sigma/delta节点保持稳定
- 活跃节点得到保护
- Temporal/Multi-hop问题中历史关键节点不被无故腐蚀

---

### Bug 3: Abstract节点稀释检索结果 ❌→✅

**文件**: `src/phaseforget/pipeline/orchestrator.py`
**方法**: `search_with_graph()`

**问题**：Abstract sigma/delta节点（只有综合信息，无具体细节）混入Top-K，淹没Multi-hop推理所需的具体事实

现象：Multi-hop从29.55→17.33暴跌

**修改**：限制abstract节点比例不超过30%

```python
# 分离 concrete 和 abstract
concrete = [r for r in results if not await is_abstract(r['id'])]
abstract = [r for r in results if await is_abstract(r['id'])]

# 只保留前30%的abstract节点
max_abs = max(1, int(len(results) * 0.3))
return concrete + abstract[:max_abs]
```

**效果**：Multi-hop恢复（从17→25），Open Domain保留概括优势但不被淹没

---

### Bug 4: 评测标准与A-MEM不统一 ❌→✅

**文件**: `src/phaseforget/evaluation/metrics.py`

**问题**：所有度量与A-MEM计算方式不同，数据无法对比

#### 4.1 F1: Counter → Set

```python
# 修改前：Counter（重复词多次计数）
common = Counter(pred_tokens) & Counter(ref_tokens)
num_common = sum(common.values())

# 修改后：Set + 标点处理（A-MEM方式）
def _simple_tokenize(text):
    text = text.lower()
    for ch in ".,!?":
        text = text.replace(ch, " ")
    return text.split()

pred = set(_simple_tokenize(prediction))
ref  = set(_simple_tokenize(reference))
common = pred & ref  # 去重
```

**影响**：参考答案有重复词时F1会升高（符合Set逻辑）

---

#### 4.2 BLEU-1: 自定义 → nltk with smoothing + brevity penalty

```python
# 修改前：无smoothing，无brevity penalty
return clipped / len(pred_tokens)

# 修改后：A-MEM方式
from nltk.translate.bleu_score import sentence_bleu, SmoothingFunction
return sentence_bleu(ref, pred, weights=(1,0,0,0),
                     smoothing_function=SmoothingFunction().method1)
# 包含 brevity_penalty = exp(1 - len(ref)/len(pred))
```

**具体例子**：
```
参考：7词，预测：1词
旧：BLEU = 1.0（满分）
新：BLEU ≈ 0.05（大幅降低）
```

**这是Open Domain从35→8的主要原因**

---

#### 4.3 ROUGE: `use_stemmer=False` → `use_stemmer=True`

```python
# 修改前
scorer = rouge_scorer.RougeScorer(["rougeL"], use_stemmer=False)

# 修改后
scorer = rouge_scorer.RougeScorer(["rougeL"], use_stemmer=True)
```

**效果**："running"和"run"现在被认为匹配，对Temporal有利

---

#### 4.4 SBERT: 0-100 → 0-1

```python
# 修改前
return sim * 100.0

# 修改后
return sim
```

**效果**：标度统一，避免数值膨胀

## 总结

| Bug | 影响 | 修复方法 |
|-----|------|---------|
| Entailment反向 | 驱逐永不发生 | 交换premise/hypothesis |
